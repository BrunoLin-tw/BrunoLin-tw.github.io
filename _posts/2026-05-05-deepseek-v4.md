DeepSeek V4 這次發布，我第一眼看的其實是它想把模型放到哪裡用，benchmark 反而排在後面。

答案很明顯：更長的 context、更便宜的推理、更靠近 agent workflow，還有一條很現實的硬體路線，華為 Ascend。

這次沒有 R1 那種突然嚇壞市場的劇情。市場已經知道中國模型會做得很便宜、很有效率。V4 比較像下一步：把模型能力、長上下文、API 成本和國產晶片整合起來，看能不能真的進入工程工作流。

## DeepSeek V4 包含哪些模型？

DeepSeek 官方這次先推出 Preview，主要是兩個版本：DeepSeek-V4-Pro 和 DeepSeek-V4-Flash。

根據 DeepSeek 官方文件和 HuggingFace model card，V4-Pro 是比較大的 MoE 模型，總參數 1.6T、啟用參數 49B，context window 是 1M tokens。它主打 coding、數學、STEM、agentic task，定位就是拿來打高難度推理和 coding agent。

V4-Flash 則是比較小、比較便宜的版本，總參數 284B、啟用參數 13B，同樣支援 1M tokens context。它的方向很清楚：速度、成本、API 大量使用。DeepSeek 也把舊的 `deepseek-chat` 和 `deepseek-reasoner` 對應到 V4-Flash 的不同模式，之後舊名稱會退場。

簡單講：

| 模型 | 定位 | 總參數 | 啟用參數 | 適合場景 |
|---|---:|---:|---:|---|
| DeepSeek-V4-Pro | 高階推理 / coding agent | 1.6T | 49B | 複雜程式分析、數學、長任務 agent |
| DeepSeek-V4-Flash | 快速 / 便宜 / 量產 API | 284B | 13B | 一般對話、批次任務、成本敏感服務 |

這個切法其實很合理。現在模型產品不能只靠一顆大模型打天下，真實產品會需要「強但貴」和「夠用但便宜」兩種路線。

## 1M token context：這比排行榜更值得看

V4 最值得注意的規格，是 1M token context。

長 context 不是新話題。DeepSeek 這次比較有意思的地方，是它把 context window 拉大，同時宣稱把長上下文的計算和記憶體成本壓下來。HuggingFace model card 提到，V4-Pro 在 1M context 場景下，相比 DeepSeek-V3.2，只需要約 27% 的 single-token inference FLOPs，以及約 10% 的 KV cache。

這對 agent 很關鍵。

Coding agent 真的跑起來時，不會只問一兩個問題。它要看 repo、讀文件、查錯誤 log、回頭比對前面的設計，甚至連續改幾輪。如果 context 很短，就會一直切片、摘要、丟失細節，最後多花很多 token 和時間補洞。

1M context 如果成本能壓住，會讓「把整個專案或大量文件丟進去」這件事更接近可用。當然，長 context 不等於模型真的理解全部內容。這點不能亂吹。它只是讓 agent 少一點健忘，多一點工作空間。

## 華為晶片訓練 / 推理：這條線比想像中重要

Reuters 和多家媒體都把 Huawei Ascend 放在這次 V4 的重點裡。華為確認 Ascend 950 supernode cluster 支援 DeepSeek V4，Reuters 也提到 V4 是針對華為晶片做適配，Flash 版本訓練流程裡也使用了部分華為晶片。

這件事要分開看。

推理端往華為 Ascend 靠，這很合理，也很實際。中國公司如果要大規模部署 AI 服務，不可能永遠把命脈押在 Nvidia 高階 GPU 上。美國出口限制已經擺在那裡，能不能把熱門開源模型跑在本土硬體上，會直接影響成本、供應鏈和部署自由度。

訓練端就不能講得太滿。大型模型訓練仍然很吃軟體生態、硬體穩定性、通訊效率，CUDA 的優勢不會因為一句「國產替代」就消失。比較務實的說法是：DeepSeek V4 代表中國 AI stack 正在把模型和本土晶片綁得更緊，尤其是推理部署這一段。

我覺得這比「V4 有沒有再震撼華爾街」更值得看。R1 是市場驚嚇，V4 像是供應鏈測試。

## 更適合 agent integration

DeepSeek 官方文件明顯把 V4 往 agent workflow 推。

它不只提供 OpenAI-compatible API，也提供 Anthropic-compatible endpoint。文件裡還直接列出 GitHub Copilot CLI、Claude Code、OpenCode、OpenClaw 這類整合方向。這很有意思，因為 DeepSeek 很清楚現在模型要進入開發者工作流，不能只停在聊天視窗。

其中一個細節很工程：DeepSeek 的 GitHub Copilot CLI 文件建議使用 Anthropic-compatible endpoint，因為 OpenAI 格式在 thinking mode 下會碰到 `reasoning_content` 必須回傳的問題。這種小地方很真實，因為 agent integration 最麻煩的常常是 API schema、tool calling、上下文格式和中途狀態能不能接得起來。

這也是我覺得 V4 值得追的原因。它沒有只賣「我比較聰明」，而是往「我可以放進 coding agent 裡跑」靠近。這才是模型真正進工作流的入口。

## 成本和效能評估

成本這段要小心，因為 DeepSeek 官方目前有折扣價，媒體引用的價格也不完全一致。

官方 pricing 頁面目前顯示：

| 模型 | Input cache miss | Output |
|---|---:|---:|
| DeepSeek-V4-Flash | US$0.14 / 1M tokens | US$0.28 / 1M tokens |
| DeepSeek-V4-Pro | US$0.435 / 1M tokens | US$0.87 / 1M tokens |

但這裡有折扣條件，V4-Pro 目前是 75% off 到 2026/05/31 UTC。部分媒體和第三方文章引用的 V4-Pro 價格是 input US$1.74、output US$3.48，比官方當前折扣價高不少。寫文章或做評估時最好看官方 pricing，不要拿單一截圖當永久價格。

效能上，DeepSeek 自己宣稱 V4-Pro 在數學、coding、STEM、agentic coding 上是開源模型前段班，部分媒體也引用它接近 GPT-5.4、Gemini 3.1 Pro、Claude Opus 4.6 這些閉源模型的說法。但我會保守一點看。

Benchmark 可以參考，不能當信仰。

對工程使用來說，我更在意四件事：

- 長 context 下是否真的穩
- agent 多輪工具使用會不會亂掉
- 實際 API 成本能不能壓住
- 在非 Nvidia 硬體上的服務品質能不能穩定

如果這四件事成立，V4 的影響會比排行榜分數更大。因為到最後，產品團隊買的不是漂亮分數，買的是少重試、少人工接手、少花冤枉 token。

## 我的看法

DeepSeek V4 這次出的招，已經不只模型變強。

它真正想打的是一整套組合：1M context、低成本推理、agent integration，再加上華為 Ascend 這條硬體路線。

這套打法很務實，也很中國 AI 目前的處境。沒有無限高階 GPU，就把架構效率、推理成本和本土硬體支援一起做。這不一定會立刻超越 OpenAI 或 Anthropic，但它會讓開源模型在真實工作流裡更有存在感。

我會把 V4 看成 DeepSeek 從「震撼市場」轉向「工程落地」的一步。

沒那麼戲劇化，但可能更重要。
